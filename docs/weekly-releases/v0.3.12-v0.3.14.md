## 本周发布更新

本周发布重点是把技能与模板分发、账号与模型管理、以及本地运行时稳定性一起补齐，Web UI 和飞书相关流程也继续细化。

### 更新内容

- OpenClaw 现已支持 Codex provider，并补上图片输入处理和更完整的流式用量信息，接入能力更接近完整的 Codex 工作流。
- 飞书集成继续扩展，新增 Agent 绑定、人类渠道管理、本地 Codex bridge 话题回复，以及更清晰的邀请失败提示和重连恢复，渠道接入与日常使用更顺。
- Web UI 补强了管理端日常操作，包括悬浮 Manager 聊天、更完整的房间成员管理、改进的 Agent 创建表单，以及更稳定一致的资料页布局。
- SkillHub 现已支持全局技能浏览、`SKILL.md` 预览、上传、删除和 Agent 级技能分配，技能分发与挂载流程更完整。
- 现已支持账号登录和集中式模型 Provider 管理，包括 OpenCSG Provider、Provider 增删改查和连通性检查，模型配置入口更统一。
- 官方模板现已切到 CSGHub 加载，并支持懒加载工作区内容和文件预览；同时内置系统技能会统一随产品提供，减少默认资源缺失或被误覆盖的情况。
- 本地数据存储已迁移到类型化 ID，并补上迁移前备份、符号链接保留和旧配置迁移，升级本地版本时更安全。
- Agent 与模板相关界面继续修正，包括恢复 Agent 档案中的加技能按钮、修复资料草稿同步、隐藏创建 Agent 时不该出现的 manager 模板，并补上更清晰的模板细节。
- Windows 与升级流程进一步补强，包括修复 Windows 上 Codex 因命令路径处理导致的不可用问题、Launcher 自动升级和官方安装升级校验，安装与升级体验更稳。
- 本地运行时稳定性继续提升，包括从宿主挂载 `csgclaw-cli` 到 sandbox、托管 `AGENTS.md` 位置修正、旧 sandbox 重建、运行时默认值、失败网关清理和飞书重连恢复。

### 涉及版本

- `v0.3.12`: 扩展 Codex 与飞书集成，改进工作区管理和多项 Web UI 流程，并收紧升级与运行时行为。
- `v0.3.13`: 新增 SkillHub 全链路管理、账号登录和集中式模型 Provider 管理，同时继续补强飞书、升级和运行时刷新流程。
- `v0.3.14`: 强化模板与系统技能分发、本地存储迁移安全，以及 Agent、模板、Windows 和运行时相关的多项可用性与稳定性修复。

## Weekly Release Update

This week’s releases focused on filling out skill and template distribution, centralizing account and model management, and hardening local runtime behavior, with continued polish across the Web UI and Feishu flows.

### What's Changed

- OpenClaw now supports the Codex provider with image input handling and richer streaming usage metadata, making Codex-backed workflows more complete.
- Feishu integration expanded with agent binding, human channel management, a local Codex bridge with threaded replies, and clearer invite-failure or reconnect handling.
- The Web UI improved day-to-day management with a floating Manager chat, stronger room member management, a cleaner agent creation form, and more consistent profile layouts.
- SkillHub now covers global skill browsing, `SKILL.md` preview, upload, deletion, and agent-scoped skill assignment, so skill distribution and attachment are managed end to end.
- Account login and centralized model provider management are now available, including built-in OpenCSG support, provider CRUD, and connectivity checks from one place.
- Official templates now load from CSGHub with lazy-loaded workspace contents and file previews, while embedded system skills ensure default skills ship consistently and are harder to overwrite accidentally.
- Local state now migrates to typed IDs with safer backups, symlink preservation, and legacy config migration, reducing upgrade risk on existing machines.
- Agent and template flows were refined by restoring the add-skill button, fixing profile draft sync, hiding manager templates during agent creation, and showing clearer template details.
- Windows and upgrade flows were improved with a fix for Codex usability on Windows caused by command path handling, plus launcher auto-upgrade and official-install upgrade checks.
- Local runtime stability improved with host-mounted `csgclaw-cli` inside sandboxes, managed `AGENTS.md` placement fixes, legacy sandbox recreation, runtime-specific defaults, failed gateway cleanup, and more reliable Feishu reconnect recovery.

### Covered releases

- `v0.3.12`: Expanded Codex and Feishu integration, improved workspace-management and several Web UI flows, and tightened upgrade or runtime behavior.
- `v0.3.13`: Added end-to-end SkillHub management, account login, and centralized model provider management, while continuing to improve Feishu, upgrade, and runtime refresh behavior.
- `v0.3.14`: Strengthened template and system-skill distribution, local-store migration safety, and a range of usability or stability fixes for agent flows, templates, Windows, and runtime handling.
