## 本周发布更新

本周发布重点是补强模型接入、模板与权限流程，并显著升级 Web UI 与 IM 协作体验。

### 更新内容

- 新增对 LLM Providers Responses API 的支持，并在不支持原生 `/responses` 时自动回退，接入不同模型服务时更稳定。
- Hub 新增远程 registry 支持，并为 agent 模板补充明确的 manager/worker 角色，模板发现、选择和运行时匹配更直接。
- 强化 manager 重建、bot 删除和相关 Web UI 入口，日常维护 agent 时更可靠，也更容易找到正确操作。
- Web UI 完成一轮较大的现代化改造，采用独立 Vite/React 应用和更清晰的工作区路由结构，浏览、切换和管理页面更顺手。
- 发布与部署流程进一步完善，发布包现在包含 Web 资源，并支持相对部署路径、sandbox 路径路由和 Dockerfile，部署环境更容易对齐。
- 通知能力调整为基于 channel bot 的中继模式，并补充对应 API、权限校验和侧边栏界面，通知接入和查看路径更完整。
- IM 侧新增 thread 支持，覆盖存储、API、bot 兼容和 Codex 会话隔离，多轮协作场景更清晰。
- 新增 Codex 工具权限审批流程，用户可以在 Web UI 中查看权限请求并逐项批准，工具执行过程更可控。
- 多项 Web UI 细节修复继续提升日常体验，包括 Hub 深色模式徽标、模板详情布局、工作区浏览样式、侧边栏品牌和 profile 控件等。

### 涉及版本

- v0.3.3: 扩展 Responses API、远程 Hub registry 和模板角色能力，并加强 manager 重建、bot 删除与相关 Web UI 交互。
- v0.3.4: 现代化重构 Web UI，完善发布与部署支持，引入 thread 与通知流程改进，并增加 Codex 工具权限审批。

## Weekly Release Update

This week's releases improve model-provider compatibility, template and permission workflows, and the overall Web UI and IM collaboration experience.

### What's Changed

- Added support for LLM provider Responses APIs, with automatic fallback when native `/responses` is unavailable, making mixed-provider integrations more reliable.
- Added remote Hub registry support and explicit manager/worker roles in agent templates, so template discovery and runtime selection are more straightforward.
- Improved manager recreation, bot deletion, and the related Web UI entry points, making routine agent maintenance more dependable.
- Modernized the Web UI with a dedicated Vite/React app and a clearer workspace routing structure, so browsing and moving through the product feels more consistent.
- Improved release and deployment packaging by shipping Web assets in release bundles and supporting deployment-relative asset paths, sandbox-path routing, and a Dockerfile.
- Moved notification workflows onto channel bots with relay support, matching API routes, permission checks, and sidebar UI, so notification delivery and visibility are better integrated.
- Added thread support across IM storage, APIs, bot compatibility, and Codex session isolation, which makes multi-step conversation workflows easier to manage.
- Added Codex tool permission approvals, allowing users to review and approve permission requests directly in the Web UI for tighter control over tool execution.
- Continued polishing the Web UI with fixes across Hub dark mode badges, template detail layout, workspace browser styling, sidebar branding, and profile controls.

### Covered releases

- v0.3.3: Expanded Responses API support, remote Hub registries, and template role handling, while improving manager recreation, bot deletion, and related Web UI flows.
- v0.3.4: Modernized the Web UI, improved release and deployment support, added thread and notification workflow updates, and introduced Codex tool permission approvals.
