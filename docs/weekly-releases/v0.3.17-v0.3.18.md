## 本周发布更新

本周发布将内置 Manager 切换到 Codex CLI runtime，并进一步补齐定时任务、Codex CLI 与 MCP 服务管理、Codex Worker 模板和 Web UI 导航体验。

### 更新内容

- 内置 Manager Agent 已切换到 Codex CLI runtime，并在 Codex 不可用时提供就绪状态提示，运行环境更清晰、更易诊断。
- 新增定时任务自动化，覆盖任务存储、调度执行、运行历史、API 和工作区管理界面，可直接管理周期性 Agent 工作流。
- 新增托管式 Codex CLI runtime，集成启动流程、runtime API 和 Computer 页面控制，Codex 的安装与管理更集中。
- Codex、OpenClaw 和 PicoClaw 现已支持托管 MCP 服务，可通过服务目录进行管理和批量安装，并保留已有配置。
- 新增内置 Codex Worker 模板，支持自动准备工作区、同步技能和检查 sandbox 可用性，创建 Codex Worker 更直接。
- 新增 GitHub Connector，提供 OAuth、配置存储和 Web UI 绑定流程，可为 Agent 接入 GitHub 上下文。
- OpenCSG 登录新增生产、Stage 和自定义站点选择，并同步适配服务端点与账号界面，方便连接不同部署环境。
- Web UI 重新设计侧边栏和页面框架，支持可定位的页面区域、持久化宽度调整，并优化设置页和响应式布局。
- Codex Manager 与 Agent 初始化流程进一步稳定，修复 Windows runtime、`codex.cmd`、下载诊断、任务标题和活动状态等问题。
- 聊天与任务体验持续优化，包括长消息折叠、自动跟随滚动、Agent 名称展示、定时任务刷新与运行反馈。

### 涉及版本

- [`v0.3.17`](https://github.com/OpenCSGs/csgclaw/releases/tag/v0.3.17): 将内置 Manager 切换到 Codex CLI runtime，新增定时任务、GitHub Connector 和 OpenCSG 多环境登录，并提升 Web UI 与 Windows 稳定性。
- [`v0.3.18`](https://github.com/OpenCSGs/csgclaw/releases/tag/v0.3.18): 新增托管式 Codex CLI runtime、MCP 服务管理和 Codex Worker 模板，重构 Web UI 导航并强化 Codex 初始化流程。

## Weekly Release Update

This week’s releases moved the built-in Manager to the Codex CLI runtime and expanded scheduled automation, managed Codex CLI and MCP setup, the Codex worker template, and Web UI navigation.

### What's Changed

- The built-in Manager agent now runs on the Codex CLI runtime and reports readiness when Codex is unavailable, making runtime status clearer and easier to diagnose.
- Scheduled task automation now covers storage, scheduler execution, run history, APIs, and workspace management, enabling recurring agent workflows to be managed directly.
- Managed Codex CLI runtimes now integrate with startup, runtime APIs, and Computer page controls, centralizing Codex installation and management.
- Codex, OpenClaw, and PicoClaw now support managed MCP servers with catalog management, batch installation, and preservation of existing configurations.
- A built-in Codex worker template now provisions workspaces, synchronizes skills, and checks sandbox availability for a more direct worker setup flow.
- A new GitHub connector adds OAuth, configuration storage, and Web UI attachment flows for bringing GitHub context into agent work.
- OpenCSG login now supports production, stage, and custom sites, with environment-aware endpoints and account UI for different deployments.
- The Web UI sidebar and page shell were redesigned with addressable sections, persistent resizing, refined settings, and improved responsive layouts.
- Codex manager and agent setup is more reliable across Windows runtimes, `codex.cmd`, download diagnostics, task titles, and activity state.
- Chat and task workflows were refined with long-message collapsing, steadier auto-follow scrolling, agent names in assignees, scheduled-task refreshes, and clearer run feedback.

### Covered releases

- [`v0.3.17`](https://github.com/OpenCSGs/csgclaw/releases/tag/v0.3.17): Moved the built-in Manager to the Codex CLI runtime, added scheduled tasks, the GitHub connector, and OpenCSG environment selection, and improved Web UI and Windows reliability.
- [`v0.3.18`](https://github.com/OpenCSGs/csgclaw/releases/tag/v0.3.18): Added managed Codex CLI runtimes, MCP server management, and a Codex worker template, redesigned Web UI navigation, and hardened Codex setup flows.
