## 本周发布更新

本周发布重点是扩展运行时能力，并把模板、Hub 和管理流程衔接得更完整。

### 更新内容

- 新增 OpenClaw 运行时支持，覆盖启动、运行时记录、API 和 Web UI，按运行时创建和管理 agent 更明确。
- 新增 Notifier 运行时支持，提供通知接口来接收 webhook，并把消息分发到 IM 渠道。
- 新增 Hub agent 模板注册、浏览、工作区预览和发布能力，agent 模板更容易发现、检查和复用。
- Web UI 统一了 agent 创建入口和 agent 模板选择流程，并改进了 Hub 导航，创建新 agent 的路径更顺畅。
- Manager 重建流程支持直接选择运行时和镜像，调整运行环境时不必再绕回手动配置。
- 内置 agent 模板与嵌入式运行时模板进一步统一，启动默认值更多从 Hub 和模板解析，初始化流程更一致。
- 多项修复提升了日常可用性，包括修复 bot 创建 agent 时的 agent 模板工作区同步、恢复 Hub agent 模板操作，以及避免运行时与模板不匹配。
- API 路由和 bot 管理入口继续整理，集成侧结构更清晰，同时旧的 IM 路由已移除并同步更新文档。

### 需要注意

- 普通用户没有明确的升级动作要求。
- 如果你维护自定义集成或自动化流程，需要注意：运行时类型现在更强调显式配置，部分旧兼容路径和旧 IM API 路由已被清理。

### 涉及版本

- v0.3.1: 引入 OpenClaw 运行时，以及 Hub agent 模板注册、浏览、预览和发布能力，并改进 agent 模板与 manager 重建流程。
- v0.3.2: 引入 Notifier 运行时，继续完善 Hub/Web UI 体验，并整理 agent、runtime 与 API 结构。

## Weekly Release Update

This week's releases expand runtime options and make templates, Hub workflows, and manager operations work together more smoothly.

### What's Changed

- Added OpenClaw runtime support across bootstrap, runtime records, APIs, and the Web UI, making runtime-specific agent creation and management clearer.
- Added Notifier runtime support, including a notification endpoint for receiving webhooks and fanning messages out into IM channels.
- Added Hub agent template registries, browsing, workspace preview, and publishing, making agent templates easier to discover, inspect, reuse, and publish.
- Unified the Web UI flow for agent creation and agent template selection, and improved Hub navigation so new agent setup is more direct.
- Restored and improved manager recreation so users can choose runtime and image directly when rebuilding a manager.
- Aligned built-in agent templates further with embedded runtime templates, and moved more bootstrap defaults onto Hub/template-driven configuration for a more consistent setup flow.
- Improved day-to-day reliability with fixes for bot-created agent template workspace sync, restored Hub agent template actions, and runtime/template mismatch prevention.
- Reorganized API routing and bot management paths, giving integrators a cleaner structure while removing legacy IM routes and updating the docs to match.

### Notes

- Normal users do not need to take any upgrade action.
- If you maintain custom integrations or automation, note that runtime kinds are now expected to be explicit, and some older compatibility paths and legacy IM API routes were removed.

### Covered releases

- v0.3.1: Added OpenClaw runtime support, introduced Hub agent template registries, browsing, preview, and publishing, and improved parts of the agent template and manager recreation flow.
- v0.3.2: Added Notifier runtime support, continued Hub/Web UI polish, and cleaned up agent, runtime, and API structure.
