## 本周发布更新

本周发布重点是完成参与者模型重构、扩展 Agent 与 Human 档案管理，并补强 Codex 运行时配置和会话恢复能力。

### 更新内容

- 完成参与者模型重构，全面替换旧的 Bot 身份表示，并补上旧参与者 ID 兼容和管理员状态迁移，团队、Agent、渠道等相关流程现在更一致。
- Web UI 新增设置保存并重启流程，常用 `config.toml` 配置可直接在页面编辑、校验并按提示重启生效。
- Agent 和 Human 现已支持端到端头像管理，包括上传、裁剪、内置头像分配，以及更安全的部分资料保存流程。
- Web UI 新增 Human 档案与私聊资料入口，头像等信息会同步回 IM 用户记录，人和 Agent 的资料管理更完整。
- Codex Agent 现已支持 instructions，并会在本地工作区生成托管的 `AGENTS.md`，方便把运行约束和协作说明带入工作区。
- Codex 运行时新增基于 schema 的配置项、本地工作区目录支持和宿主目录选择器，配置入口更清晰，工作区选择也更直接。
- 团队与任务工作流继续优化，包括参与者映射重构、任务执行逻辑下沉到 `internal/team`，以及任务视图中的 Agent 展示名支持。
- Web UI 继续打磨日常交互，包括更顺手的 slash 命令选择、更准确的话题工具调用展示、升级后详情刷新，以及 Manager 初始化提示与工作区刷新。
- 飞书凭据现已通过参与者统一绑定，CLI、API、Provider 和相关文档的配置方式更加一致。
- Codex 运行时进一步补强稳定性，包括服务重启后恢复会话、注入环境变量并按 JSON 流解析 app-server 输出，以及仅在工作区目录变更时触发重启。

### 涉及版本

- `v0.3.10`: 完成参与者模型替换并补强迁移，新增设置保存重启流程，继续优化团队任务流、Web UI 日常交互，以及 Codex 运行时布局和会话恢复。
- `v0.3.11`: 新增 Agent 与 Human 头像和档案管理、Codex Agent instructions 与托管 `AGENTS.md`、schema 驱动的运行时配置与目录选择器，并统一飞书凭据绑定方式。

## Weekly Release Update

This week’s releases focused on finishing the participant-model refactor, expanding profile management for agents and humans, and making Codex runtime configuration or session recovery more practical.

### What's Changed

- The platform now completes the participant-model refactor end to end, replacing legacy bot identities and adding follow-up fixes for old participant IDs and admin-state migration.
- The Web UI now includes a save-and-restart settings flow so common `config.toml` fields can be edited, validated, and applied from the UI.
- Agents and humans now support end-to-end avatar management, including upload, cropping, built-in avatar assignment, and safer partial-profile saves.
- The Web UI now adds human profiles and direct-message profile actions, with avatar changes synced back to IM user records.
- Codex agents now support instructions and generate managed `AGENTS.md` content inside local workspaces, making workspace guidance easier to carry with the agent.
- Codex runtime setup now uses schema-driven options, supports local workspace directories, and adds a host directory picker for clearer workspace configuration.
- Team and task workflows were refined with participant mapping cleanup, task execution logic moved into `internal/team`, and friendly agent display names in task views.
- Daily Web UI behavior was polished with smoother slash-command selection, more accurate thread tool-call rendering, refreshed upgrade details, and better manager setup or workspace refresh handling.
- Feishu credentials are now bound through participants across the CLI, API, provider wiring, and related documentation.
- Codex runtime resilience improved with restored sessions after server restart, injected runtime env handling plus JSON-stream stdout decoding, and narrower restarts that trigger only on workspace directory changes.

### Covered releases

- `v0.3.10`: Completed the participant-model migration, added the save-and-restart settings flow, and continued improving team task workflows, day-to-day Web UI behavior, and Codex runtime layout or session recovery.
- `v0.3.11`: Added avatar and profile management for agents and humans, Codex agent instructions with managed `AGENTS.md`, schema-driven runtime configuration with directory picking, and unified Feishu credential binding.
