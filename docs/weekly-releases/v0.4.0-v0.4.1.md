## 本周发布更新

本周发布重点为 Manager 增加文件上传和图片识别，增强聊天交互与会话可观测性，并提升 OpenCSG 集成、Manager 重建、应用内升级和 Web UI 工作流的可靠性。

### 更新内容

- Manager 聊天现已支持文件上传；使用兼容 Responses API 的模型时，还可直接识别用户发送的图片，并增强了 Codex 会话恢复能力。
- 新增实时会话活动面板，支持筛选、排序和查看事件详情，同时覆盖私聊与房间会话。
- Agent 可在聊天中发起交互式问题，用户能够直接回答或取消，让需要确认的任务无需跳出会话。
- Codex Manager 会在 MCP 配置变更后自动重建，并通过清理重试和 NFS 竞态防护提升重建可靠性。
- OpenCSG 登录后会同步环境和模型信息，并支持托管登录回调及按环境获取资源端点，减少手动配置。
- CLIProxy 已支持最新 Codex 模型，CSGHub Worker 同时限定使用 Codex，避免创建不兼容的运行配置。
- 应用内升级支持由 CSGHub 托管重启，并在升级完成后自动刷新页面，让升级状态和后续操作更明确。
- Agent Profile 元数据支持自动保存，关闭 Agent 抽屉或从详情页进入私聊时也会正确处理未保存的修改，减少编辑内容意外丢失。
- Web UI 优化了团队与 Manager 头像、模型配置、会话计数、长消息控制、导航、状态提示和暗色模式等细节。

### 涉及版本

- `v0.4.0`: Manager 新增文件上传及基于 Responses API 兼容模型的图片识别，同时支持最新 Codex 模型，并完善 OpenCSG 登录、托管升级和 Web UI 体验。
- `v0.4.1`: 新增会话活动面板和交互式 Agent 问答，并增强 Manager 重建、模型同步、Profile 编辑与升级流程的可靠性。

## Weekly Release Update

This week’s releases added file uploads and image recognition to Manager, improved chat interaction and conversation observability, and made OpenCSG integration, Manager recreation, in-app upgrades, and Web UI workflows more reliable.

### What's Changed

- Manager chat now supports file uploads and can recognize uploaded images when using a Responses API-compatible model, together with stronger Codex session recovery.
- A real-time conversation activity panel adds filtering, sorting, and event details for both direct messages and rooms.
- Agents can ask interactive questions in chat, where users can answer or cancel without leaving the conversation.
- Codex Manager now recreates automatically after MCP changes, with cleanup retries and NFS race protection for more reliable recovery.
- OpenCSG login now synchronizes environments and models, supports hosted callbacks, and derives resource endpoints from the selected environment to reduce manual setup.
- CLIProxy supports the latest Codex models, while CSGHub workers are restricted to Codex to prevent incompatible runtime configurations.
- In-app upgrades now support CSGHub-managed restarts and refresh the page after completion for a clearer end-to-end upgrade flow.
- Agent profile metadata now autosaves, while closing the agent drawer or opening a direct message from agent details correctly handles unsaved changes to prevent accidental edit loss.
- The Web UI improves team and Manager avatars, model configuration, conversation counts, long-message controls, navigation, status indicators, and dark-mode presentation.

### Covered releases

- `v0.4.0`: Added file uploads and image recognition with Responses API-compatible models to Manager, supported the latest Codex models, and refined OpenCSG login, managed upgrades, and the Web UI.
- `v0.4.1`: Added the conversation activity panel and interactive agent questions, and improved Manager recreation, model synchronization, profile editing, and upgrade reliability.
