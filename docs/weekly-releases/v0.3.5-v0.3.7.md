## 本周发布更新

本周发布重点围绕升级体验、话题与技能流程、运行时可见性，以及 Web UI 的持续打磨。

### 更新内容

- 新增 PicoClaw 话题支持，支持在话题中提及成员，回复也会保持在话题内，不干扰主房间时间线。
- 新增 `skill` CLI 命令，支持搜索、安装、查看版本和解析 ClawHub 技能来源，便于管理扩展能力。
- Hub 模板中的环境变量会在创建 Agent 时自动应用，模板化配置开箱即用程度更高。
- 管理型部署现在可以通过 `show_upgrade` 配置隐藏升级入口，避免不应自升级的环境误触升级动作。
- IM 会将超大 session 消息自动转存为 blob，减少因大小限制导致的会话可读性问题。
- PicoClaw runtime 的工作区目录布局已与 OpenClaw 对齐，sandbox 和 gateway 行为更一致。
- Web UI 持续优化了按钮、表单校验、消息结构、消息气泡、侧边栏交互、版本展示、升级弹窗和升级菜单，日常操作更一致也更易读。
- 本地构建和开发版本不再显示误导性的升级提示，同时仍会保留最新版本信息展示。
- 前台启动场景下，升级完成后会提示手动重启，避免升级完成后状态不明确。
- 运行时状态更容易排查，Agent 启动就绪状态和日志现在可以直接查看。
- 默认 server base URL 的解析逻辑已集中处理，同时改进了本机 IP 探测，减少本地运行时的配置摩擦。
- 修复了 Manager 和话题相关的行为回归，减少异常的重建标记和话题展示偏差。

### 涉及版本

- `v0.3.5`: 增加前台启动场景下升级后的手动重启提示，支持超大 IM 消息转存，并让 PicoClaw runtime 工作区布局与 OpenClaw 保持一致。
- `v0.3.6`: 增加 PicoClaw 话题能力与 `skill` CLI 命令，同时修正本地构建升级提示、默认 URL 解析、本机 IP 探测以及部分 Manager/话题行为问题。
- `v0.3.7`: 增加运行时就绪状态与日志查看、模板环境变量注入，以及可控制升级入口显示的服务端配置。

## Weekly Release Update

This week’s releases focused on smoother upgrades, better topic and skill workflows, stronger runtime visibility, and steady Web UI polish.

### What's Changed

- PicoClaw now supports threaded conversations, including mentions in threads and replies that stay out of the main room timeline.
- A new `skill` CLI command adds search, install, version, and registry resolution flows for ClawHub-managed skills.
- Hub template environment variables are applied during agent creation, so template-based setup is more complete out of the box.
- Managed deployments can now hide upgrade actions with the `show_upgrade` server setting when self-upgrade should not be exposed.
- Oversized IM session messages can spill into blobs, helping sessions stay readable under message size limits.
- The PicoClaw runtime workspace layout now matches the OpenClaw structure more closely for more consistent sandbox and gateway behavior.
- The Web UI was refined across buttons, form validation, message structure, message bubbles, sidebar behavior, version display, the upgrade modal, and the upgrade menu for a more consistent daily workflow.
- Local or development builds no longer show misleading upgrade nags, while still reporting the latest release version.
- Foreground startup flows now prompt for a manual restart after an upgrade so the post-upgrade state is clearer.
- Agent startup is easier to inspect because runtime readiness and logs are now visible in the product.
- Default server base URL handling is now centralized, and local IP detection was improved to reduce local runtime and config friction.
- Manager and thread behavior regressions were fixed to avoid incorrect recreation flags and thread display inconsistencies.

### Covered releases

- `v0.3.5`: Added post-upgrade restart guidance for foreground startup flows, introduced oversized IM message spillover, and aligned the PicoClaw runtime workspace layout with OpenClaw.
- `v0.3.6`: Added PicoClaw topic support and the `skill` CLI command, while fixing local-build upgrade prompts, default URL resolution, local IP detection, and manager or topic behavior regressions.
- `v0.3.7`: Added runtime readiness and log visibility, applied template environment variables during agent creation, and introduced server-side control over upgrade action visibility.
