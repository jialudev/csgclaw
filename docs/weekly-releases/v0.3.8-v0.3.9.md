## 本周发布更新

本周发布重点是扩展 OpenClaw 集成、补强团队与 Agent 工作流，并继续打磨 Web UI 和 Codex 运行时稳定性。

### 更新内容

- OpenClaw 新增飞书渠道支持，和现有 PicoClaw 飞书接入能力保持一致，便于统一渠道接入。
- Slash 命令技能流程继续补强，支持更规范的调用方式、私聊中的 slash 选择，以及基于工作区的预览体验。
- Web UI 新增工作区文件预览，并改进聊天时间戳、日期分隔、本地化标签、工具提示和侧边栏细节，日常浏览与操作更顺手。
- 支持基于模板创建 Worker，并支持点号格式的 Hub 模板引用和本地模板删除，模板管理与复用更完整。
- Agent 现在支持运行时镜像升级，并修复删除 Agent 前先停止 sandbox 的清理流程，减少升级和回收过程中的异常。
- Agent 头像已打通模型、服务、Bot 创建和 Web UI 展示，便于更快识别不同 Agent。
- 团队任务执行和任务看板交互继续优化，补上更清晰的规划、任务房间投影、父子任务细节和启动控制。
- 所有运行时现在都支持 `/new` 会话重置，同时 IM 房间支持清空消息，重新开始对话更直接。
- Web UI 进一步优化了 Agent 详情操作、聊天浮层层级，以及 slash tokenization 后输入光标保持，减少编辑和操作中的干扰。
- Codex 运行时集成和本地工作区行为更稳，包括稳定 provider bridge、隔离运行时 shell 与宿主启动环境，以及保留 bind mount 工作区归属信息。

### 涉及版本

- `v0.3.8`: 增加 OpenClaw 飞书渠道、slash 技能调用与工作区预览、工作区文件预览、模板化 Worker 创建、本地模板删除，以及运行时镜像升级和更稳的 Agent 清理流程。
- `v0.3.9`: 增加 Agent 头像、跨运行时 `/new` 会话重置和 IM 清空消息能力，并继续优化团队任务流、Web UI 交互，以及 Codex 运行时和本地工作区稳定性。

## Weekly Release Update

This week’s releases focused on broader OpenClaw integrations, stronger team and agent workflows, and continued polish for the Web UI and Codex runtime stability.

### What's Changed

- OpenClaw now supports the Feishu channel, bringing it in line with the existing PicoClaw Feishu integration for more consistent channel rollout.
- Slash-command skill flows were expanded with cleaner invocation handling, slash selection in direct messages, and workspace-backed previews.
- The Web UI now includes workspace file previews and improves chat timestamps, date dividers, localized labels, tooltips, and sidebar details for smoother daily use.
- Workers can now be provisioned from templates, while dotted Hub template references and local template deletion make template reuse and cleanup more complete.
- Agents now support runtime image upgrades, and agent deletion cleanup is safer because sandboxes are stopped before agent homes are removed.
- Agent avatars now work end to end across models, services, bot creation, and the Web UI, making agents easier to identify.
- Team task execution and board interactions were improved with clearer planning, task-room projection, parent-child task details, and start controls.
- `/new` conversation reset now works across runtimes, and IM rooms can clear messages when users need a clean restart.
- The Web UI further improved agent detail actions, chat overlay layering, and composer caret preservation after slash tokenization.
- Codex runtime integration and local workspace handling were hardened by stabilizing the provider bridge, isolating runtime shells from host startup environments, and preserving ownership for bind-mounted workspaces.

### Covered releases

- `v0.3.8`: Added OpenClaw Feishu support, stronger slash skill flows and workspace previews, workspace file preview, template-based worker provisioning, local template deletion, plus runtime image upgrades and safer agent cleanup.
- `v0.3.9`: Added agent avatars, cross-runtime `/new` reset, and IM message clearing, while continuing to improve team task flows, Web UI behavior, and Codex runtime or local workspace stability.
